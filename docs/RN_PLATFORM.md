# docs/RN_PLATFORM.md — RN + Expo rules

The non-obvious things to know about this stack. Read before touching
native config, Metro, Hermes, or EAS.

## Expo managed workflow

- **No `ios/` / `android/` in git.** They're generated by `expo
  prebuild`; edits there are silently lost on the next prebuild.
- **Native config → `app.json` + plugins.** Never edit AndroidManifest
  or Info.plist directly.
- **Native modules:** check the SDK 54 list at
  https://docs.expo.dev/versions/v54.0.0/. Anything else needs a config
  plugin or a dev build. Log the decision in `DECISIONS.md`.

## Metro

- Metro's cache rarely lies — when a change doesn't appear, the file
  probably isn't in its watched paths. Check `metro.config.js` before
  `--reset-cache`.
- `require` cycles break on Hermes at runtime, not compile time. Lint
  rule `import/no-cycle` is on for this reason.

## Hermes

- Default engine on iOS + Android. Limited `Intl` support — verify i18n
  on a device, not just web.
- Release stack traces are obfuscated unless sourcemaps are uploaded to
  EAS.

## File system

- No Node `fs`. Use `expo-file-system` for app data.
- `require('./features/ui-home.json')` works at bundle time → frozen object.

## Lists

- Use `FlatList` or `@shopify/flash-list` for any list >20 items. Don't
  `.map` over `<View>`s — every cell mounts synchronously.

## Reanimated

- `react-native-reanimated` v4 + `react-native-worklets` are installed.
  Worklets must start with `'worklet'`. Call JS from a worklet via
  `runOnJS` or it crashes silently in release.

## EAS Build

- Builds go through `eas build`. No signing material in git — use EAS
  Secrets.

## When platform-touching, it's a new feature

Adding a dep, changing `app.json`, adding a config plugin → log it as
a new file in `features/`, not a side-edit on the current feature.
